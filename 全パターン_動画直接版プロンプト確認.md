# 動画直接版プロンプト — 全5パターン実出力確認

各パターンで「実際にGeminiに送られるプロンプト文字列」を展開しています。
swing-analyzer は `parts = [動画1(, 動画2), テキスト(=このプロンプト)]` の形で Gemini に送ります。

## パターン1: 自分解析（単体） — `プロンプト_自分解析_単体_動画直接_()`

**Geminiへの入力**: [ユーザーの動画] + 以下のテキスト

```
あなたはPGAツアープロを指導するトップレベルのゴルフコーチだ。
温かく褒めるところはしっかり褒めつつ、直すべきところは具体的に的確に伝える。
「ここは本当にいいね！」「自信を持っていい」といったポジティブな声かけを積極的にする。
ただし嘘はつかない。良くない部分は「ここを直せばもっと良くなる」という前向きな言い方で伝える。
絵文字は適度に使う。

【言葉選びのルール（重要）】
・ゴルフ初心者〜中級者が読むことを前提に、できるだけ平易な日本語で書く。
・専門用語を使う場合は、必ずその場で簡単な説明を添える。
  例：「レイドオフ（トップでクラブが左を向きすぎる状態）」
  例：「キャスティング（手首の角度が早く解けること）」
・「インサイドに引きすぎ」ではなく「クラブを体の内側に引きすぎている」のように、動作を具体的に描写する。

【評価ルール】

・直した方がいい点は「現状どうなっているか」「それがなぜ問題か」「どうすれば良くなるか」を分かりやすく書く。
・本当に問題がないフェーズでも、「さらに良くするためのヒント」を1つ添える。「特段の問題なし」とだけ書くのは禁止。
```

## 気になる点・要確認事項

### 1. `_解析テキストガイド_` の文言が動画直接版に不適切

自分解析・プロ比較・過去比較のベース関数内に以下の文言が含まれている:
> 「解析結果テキストはAIによる自動解析であり、同じ動画でも毎回異なる表現・観察が出る場合がある」
> 「解析テキストの表現が極端にポジティブ...」

旧方式ではStage 1の生テキストをStage 2のAIが読むので、この注意書きは意味があった。
しかし動画直接版ではGeminiが動画を直接見るので、「解析テキストの読み方」は不要。
→ **この部分は動画直接版では無害だが、やや混乱を招く可能性がある。**
→ 現時点では既存ベース関数を再利用しているので仕方ないが、将来的には動画直接版専用のベース関数を作って `_解析テキストガイド_` を除外した方がクリーン。

### 2. プロ比較・過去比較のベース関数に「解析結果を渡す」という文言がある

例: `_プロンプト_プロ比較_ベース_()` の冒頭に:
> 「これからプロと一般ゴルファーの2つのスイング解析結果を渡す。」

動画直接版では「解析結果」ではなく「動画」を渡すので、この文言は正確ではない。
→ 現時点ではGeminiが文脈から理解できるのでエラーにはならないが、将来的に修正が望ましい。

### 3. 質問モードに用語集がない

他の4パターンには `_用語集指示_`（📖 用語集セクション）が含まれているが、質問モードには含まれていない。
→ 質問モードは自由形式なので、あえて入れていない設計。問題なければこのままでOK。
