# 指示: gas/70_プロンプト.js に動画直接版プロンプト関数を追加

## 状況

`gas/60_worker（自動解析）.js` はすでに改修済みで、以下5つの関数を呼んでいる。
しかし `gas/70_プロンプト.js` にこれらの関数がまだ存在しない。追加してほしい。

呼ばれている関数名:
1. `プロンプト_自分解析_単体_動画直接_()`
2. `プロンプト_自分解析_テキストあり_動画直接_(userMessage)`
3. `プロンプト_比較_動画直接_(userMessage)`
4. `プロンプト_過去比較_動画直接_(userMessage)`
5. `プロンプト_質問モード_動画直接_(userMessage)`

## 旧方式と新方式の違い（理解してから作業すること）

旧方式（text-answer経由・2段階）:
- 1回目のAIが動画を見て「生テキスト（観察メモ）」を作る
- 2回目のAIが、その生テキスト + プロンプトを受け取って整形回答を作る
- プロンプト関数は `基本解析`（=生テキスト）を引数で受け取り、プロンプト内に埋め込んでいた

新方式（動画直接・1段階）:
- AIが動画を直接見て、プロンプトに従って1回で整形回答を返す
- 「生テキスト」は存在しない。動画そのものがAIに渡される
- プロンプト関数は `基本解析` を受け取らない。「添付された動画を見て評価してください」という指示を書く

## 重要: 既存ベース関数をそのまま再利用してはいけない

既存のベース関数（`_プロンプト_自分解析_ベース_()` 等）には、旧方式向けの以下の文言が含まれている:

1. `_解析テキストガイド_`（「解析結果テキストはAIによる自動解析であり…表現に引きずられないで…」）
   → 新方式ではAIが自分で動画を見るので、他のAIが書いたテキストを読む工程がない。この注意書きは不要。

2. プロ比較・過去比較ベースの冒頭の「これから…2つのスイング**解析結果**を渡す」
   → 新方式では解析結果テキストではなく**動画そのもの**を渡す。「動画」と書くべき。

したがって、**動画直接版では専用のベース関数を新規作成する**こと。既存のベース関数の内容をコピーした上で、以下を修正する:

- `_解析テキストガイド_` の参照を削除する
- 「解析結果を渡す」→「スイング動画を渡す」に変更する
- それ以外（コーチ冒頭、評価ルール、遵守事項、出力フォーマット、用語集指示）はそのまま維持する

## 追加する関数の仕様

### 追加場所
`gas/70_プロンプト.js` のファイル末尾に追加する。既存の関数は一切変更しない。

### セクションヘッダー
```javascript
/* =========================================================
 * 動画直接版プロンプト関数
 * swing-analyzer に動画+プロンプトを一括で渡す方式用。
 * 60_worker.js から呼ばれる。
 * text-answer（2段階目）は使わない。
 * =======================================================*/
```

### 関数1: `プロンプト_自分解析_単体_動画直接_()`

- 専用ベース関数 `_プロンプト_自分解析_動画直接_ベース_()` を新規作成
- `_プロンプト_自分解析_ベース_()` の内容をコピーして以下を修正:
  - `_解析テキストガイド_` の行を削除
- 公開関数の末尾は:
```
添付されたスイング動画を直接分析し、評価ルールに従って各フェーズを評価してください。
```

### 関数2: `プロンプト_自分解析_テキストあり_動画直接_(userMessage)`

- 専用ベース関数 `_プロンプト_自分解析_補足_動画直接_ベース_()` を新規作成
- `_プロンプト_自分解析_補足ベース_()` の内容をコピーして以下を修正:
  - `_解析テキストガイド_` の行を削除
- 公開関数の末尾は:
```
【ユーザーからのメッセージ】
「{userMessage}」

上記のメッセージの内容に応じて柔軟に対応してください。
質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に分析してください。
その上で、添付されたスイング動画を直接分析し、通常の解析フォーマットで出力してください。
```

### 関数3: `プロンプト_比較_動画直接_(userMessage)`

- 専用ベース関数2つを新規作成:
  - `_プロンプト_プロ比較_動画直接_ベース_()`
  - `_プロンプト_プロ比較_補足_動画直接_ベース_()`
- それぞれ `_プロンプト_プロ比較_ベース_()` / `_プロンプト_プロ比較_補足ベース_()` をコピーして以下を修正:
  - `_解析テキストガイド_` の行を削除
  - 「これからプロと一般ゴルファーの2つのスイング解析結果を渡す。」→「これからプロと一般ゴルファーの2つのスイング動画を渡す。」
  - 「プロの解析結果から読み取れるお手本の動き」→「プロの動画から読み取れるお手本の動き」
  - 「自分の解析結果から読み取れる現状」→「自分の動画から読み取れる現状」
- userMessageが空かどうかで補足ベース/通常ベースを切り替える（既存の `プロンプト_比較_()` と同じロジック）
- 公開関数の末尾は:
```
添付された2本のスイング動画を直接分析してください。
1本目がプロのお手本、2本目が私のスイングです。
比較してください。
```
- userMessageがある場合は追加:
```
【ユーザーからのメッセージ】
「{userMessage}」

上記のメッセージの内容に応じて柔軟に対応してください。
質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に比較してください。
その上で、通常の比較フォーマットで出力してください。
```

### 関数4: `プロンプト_過去比較_動画直接_(userMessage)`

- 専用ベース関数2つを新規作成:
  - `_プロンプト_過去比較_動画直接_ベース_()`
  - `_プロンプト_過去比較_補足_動画直接_ベース_()`
- それぞれ `_プロンプト_過去比較_ベース_()` / `_プロンプト_過去比較_補足ベース_()` をコピーして以下を修正:
  - `_解析テキストガイド_` の行を削除
  - 「これから同じゴルファーの「過去のスイング」と「今回のスイング」の2つの解析結果を渡す。」→「これから同じゴルファーの「過去のスイング」と「今回のスイング」の2本の動画を渡す。」
  - 「過去の解析結果から読み取れる当時の動き」→「過去の動画から読み取れる当時の動き」
  - 「今回の解析結果から読み取れる現状」→「今回の動画から読み取れる現状」
- 公開関数の末尾は:
```
添付された2本のスイング動画を直接分析してください。
1本目が過去の私のスイング、2本目が今回の私のスイングです。
```
- userMessageがある場合は追加:
```
【ユーザーからのメッセージ】
「{userMessage}」

上記のメッセージの内容に応じて柔軟に対応してください。
質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に比較してください。
その上で、通常の過去比較フォーマットで出力してください。
```

### 関数5: `プロンプト_質問モード_動画直接_(userPrompt)`

これは他のモードとは**完全に別構造**。フェーズ別評価は含めない。用語集も含めない。
既存のベース関数は使わず、独立した関数として作る。

```javascript
function プロンプト_質問モード_動画直接_(userPrompt) {
  return [
    "あなたは世界No.1のプロのゴルフコーチです。",
    "ユーザーから添付されたスイング動画と質問が送られてきます。",
    "",
    "【回答ルール】",
    "・動画を直接見て、ユーザーの質問に丁寧かつ的確に回答する",
    "・フェーズ別評価のフォーマットは使わない。質問に対する自由形式の回答にする",
    "・動画で確認できる内容を中心にアドバイスする（見えないことは断定しない）",
    "・改善のための具体的なアドバイスやドリルがあれば積極的に教える",
    "・専門用語を使う場合は初心者にも分かるよう補足する",
    "・絵文字は適度に使ってOK",
    "",
    "【ユーザーからの質問】",
    String(userPrompt || ""),
    "",
    "添付されたスイング動画を見た上で、上記の質問に回答してください。",
  ].join("\n");
}
```
